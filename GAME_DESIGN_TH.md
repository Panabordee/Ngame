# Cipher Deck — เกมไพ่ Deduction (เอกสารออกแบบ v0.1)

> ประเภท: เกมไพ่ multiplayer แนว deduction / bluffing ที่มีกติกาและอัตลักษณ์แบบไพ่เป็นของตนเอง

## 1. Core loop

- สำรับประกอบด้วยไพ่มาตรฐาน 52 ใบและ Joker ที่เซิร์ฟเวอร์สุ่มจำนวน 2–4 ใบ รวม 54–56 ใบต่อเกม
- รองรับผู้เล่น 3–6 คน ผู้เล่นแต่ละคนมี rack ส่วนตัว ไพ่ปกติต้องเรียงจากน้อยไปมากตามหน้า (`A, 2–10, J, Q, K`) และถ้าหน้าเท่ากันให้สีแดงอยู่ก่อนสีดำ ไม่ใช้ดอกในการเรียงหรือเดา ไพ่หน้าและสีเดียวกันถือว่าเท่ากันในการเรียง ส่วน Joker วางตำแหน่งใดก็ได้และไม่ถูกนำมาคิดตอนตรวจลำดับไพ่ปกติ
- ตอนเริ่มเกม เซิร์ฟเวอร์สุ่มจำนวน Joker สับสำรับ และแจกโดยต้องเหลือไพ่ให้จั่วครบอย่างน้อย 4 รอบ:
  - `จำนวนไพ่พื้นฐาน = min(8, floor((ขนาดสำรับ - 4 × จำนวนผู้เล่น) / จำนวนผู้เล่น))`
  - ผู้เล่นคนแรกได้ `พื้นฐาน + 1` ใบ
  - ผู้เล่นคนสุดท้ายได้ `พื้นฐาน - 1` ใบ
  - ผู้เล่นคนอื่นได้เท่ากับจำนวนพื้นฐาน
- ผู้เล่นเห็นไพ่ของตนเองทั้งหมด แต่เห็นค่าไพ่ฝ่ายตรงข้ามเฉพาะใบที่ถูกเปิดแล้ว
- ในเทิร์นของตน ถ้ากองจั่วยังมีไพ่ให้จั่วหนึ่งใบ วางใน rack ที่ตำแหน่งถูกต้อง แล้วเดาไพ่ฝ่ายตรงข้ามหนึ่งใบ ไพ่ปกติเดาด้วยหน้าและสี ส่วน Joker เดาด้วย `JOKER` โดยไม่มีสี
  - เดาถูก → เปิดไพ่เป้าหมายถาวรและเดาต่อได้
  - เดาผิดหลังจั่ว → เปิดไพ่ที่เพิ่งจั่วในตำแหน่งที่เรียงถูกต้อง แล้วจบเทิร์นทันที
  - เดาผิดเมื่อกองจั่วหมด → เลือกไพ่คว่ำของตนเองหนึ่งใบเพื่อเปิดถาวร แล้วจบเทิร์นทันที
- Joker คือไพ่ Cipher ที่วางตรงไหนก็ได้ การเดา Joker ถูกจะเปิดเฉพาะ Joker ใบที่เลือกและให้เดาต่อเหมือนไพ่ปกติ
- ผู้เล่นแพ้เมื่อไพ่ทุกใบใน rack ถูกเปิด ผู้เล่น active คนสุดท้ายเป็นผู้ชนะ

## 2. Lobby และการเชื่อมต่อกลับ

- ห้องกำหนด `desiredPlayers` ตายตัวตั้งแต่ 3–6 และเริ่มอัตโนมัติเมื่อผู้เล่นที่ยืนยันตัวตนครบ
- ล็อกห้องเมื่อเกมเริ่มและไม่รับผู้เล่นกลางเกม
- เมื่อผู้เล่นหลุด ให้หยุด action ทั้งห้องและรอเชื่อมต่อกลับ 30 วินาที
- ถ้าไม่กลับภายในเวลา ให้เปิดไพ่ทั้งหมดของผู้เล่นคนนั้นและแพ้โดย forfeit ถ้ามีไพ่ที่จั่วแต่ยังไม่วาง ต้องนำไปวางและเปิดก่อนเพื่อไม่ให้ไพ่หายจาก authoritative state
- ถ้าเป็นเทิร์นของผู้เล่นที่ forfeit ให้เลื่อนไปผู้เล่น active คนถัดไป และตรวจเงื่อนไขผู้ชนะทุกครั้ง

## 3. แนวทางต่อยอด

- เพิ่มไพ่ Wild/Cipher ที่มีความสามารถพิเศษ
- เพิ่มตัวจับเวลาในเทิร์น
- เพิ่มโหมดทีม 2v2 พร้อม hint token ร่วมกัน
- เพิ่ม ranked/MMR matchmaking

## 4. Tech stack

### Phase 1 — Web multiplayer

- Frontend: React + TypeScript + Vite
- Realtime: Node.js + Colyseus แบบ authoritative server
- Backend สำหรับ auth/leaderboard: FastAPI
- ฐานข้อมูล: PostgreSQL และ Redis
- Deploy: Docker Compose บน Proxmox/Docker และ reverse proxy ภายนอก

### Phase 2 — Desktop

- ห่อ client ด้วย Electron หรือสร้างใหม่ด้วย Godot 4
- เพิ่ม Steamworks หลัง web core loop พร้อมใช้งานเท่านั้น
- Realtime backend และ protocol เดิมยังใช้ต่อได้

## 5. Roadmap โดยย่อ

1. ทดสอบ local ด้วย browser 3–6 session
2. เชื่อม Colyseus ผ่าน LAN และ internet
3. ทำ matchmaking, match history และ leaderboard
4. ขัดเกลา UI/UX, เสียง และ animation
5. ทำ desktop build และทดสอบ platform integration
