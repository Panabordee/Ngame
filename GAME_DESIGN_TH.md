# Cipher Deck — เกมไพ่ Deduction (เอกสารออกแบบ v0.1)

> ประเภท: เกมไพ่ multiplayer แนว deduction / bluffing ที่มีกติกาและอัตลักษณ์แบบไพ่เป็นของตนเอง

## 1. Core loop

- ห้องเลือกกติกาได้ระหว่าง Classic และ Custom โดย Classic ใช้ไพ่มาตรฐาน 52 ใบและ Joker ที่เซิร์ฟเวอร์สุ่ม 2–4 ใบ รวม 54–56 ใบ ส่วน host ของห้องแบบรหัสเลือก Custom ได้ตั้งแต่ไพ่รวม 24–56 ใบ, Joker 2–4 ใบ และสำรองรอบจั่ว 1–8 รอบ เซิร์ฟเวอร์สุ่มเลือกไพ่มาตรฐานตามจำนวน ใส่ Joker ให้ครบ แล้วสับอย่างปลอดภัย
- รองรับผู้เล่น 3–6 คน ผู้เล่นแต่ละคนมี rack ส่วนตัว ไพ่ปกติต้องเรียงจากน้อยไปมากตามหน้า (`A, 2–10, J, Q, K`) และถ้าหน้าเท่ากันให้สีแดงอยู่ก่อนสีดำ ไม่ใช้ดอกในการเรียงหรือเดา ไพ่หน้าและสีเดียวกันถือว่าเท่ากันในการเรียง ส่วน Joker วางตำแหน่งใดก็ได้และไม่ถูกนำมาคิดตอนตรวจลำดับไพ่ปกติ
- เมื่อทุกคน ready แล้ว host เป็นผู้กดเริ่ม เซิร์ฟเวอร์แสดงไพ่คว่ำ 6 ใบให้ผู้เล่นที่มีสิทธิ์เลือกคนละใบไม่ซ้ำ แล้วเปิดไพ่ที่เลือกพร้อมกัน ไพ่หน้าสูงสุดเป็นคนเริ่มและ Joker สูงกว่า `K` ถ้าหน้าสูงสุดเสมอกันหรือได้ Joker หลายคน ให้เฉพาะคนที่เสมอกันเลือกใหม่จากไพ่สุ่มชุดใหม่ 6 ใบที่ไม่ใช่ชุดก่อน ทำซ้ำจนเหลือผู้เริ่มหนึ่งคน
- ไพ่ใบสุดท้ายที่แต่ละคนเลือกจะเข้า rack แบบเปิดและนับรวมในไพ่เริ่มต้น ไพ่ปกติถูกใส่ในตำแหน่งที่ถูกโดยอัตโนมัติ ก่อนเทิร์นแรก เจ้าของต้องเลือกตำแหน่งให้ Joker ทุกใบในมือเริ่มต้น ไม่ว่าจะเป็น Joker คว่ำที่ได้รับจากการแจกปกติหรือ Joker เปิดที่เลือกตอนหาคนเริ่ม โดยคู่แข่งจะไม่รู้ว่าไพ่คว่ำใบใดเป็น Joker
- เซิร์ฟเวอร์แจกโดยเหลือไพ่ให้จั่วตามจำนวนรอบที่ห้องกำหนด (Classic ใช้ 4 รอบ):
  - `จำนวนไพ่พื้นฐาน = min(8, floor((ขนาดสำรับ - รอบจั่ว × จำนวนผู้เล่น) / จำนวนผู้เล่น))`
  - ผู้เล่นที่สุ่มได้เริ่มได้ `พื้นฐาน + 1` ใบ
  - ผู้เล่นก่อนหน้าคนเริ่มในลำดับเทิร์นได้ `พื้นฐาน - 1` ใบ
  - ผู้เล่นคนอื่นได้เท่ากับจำนวนพื้นฐาน
- ผู้เล่นเห็นไพ่ของตนเองทั้งหมด แต่เห็นค่าไพ่ฝ่ายตรงข้ามเฉพาะใบที่ถูกเปิดแล้ว
- ถ้ากองจั่วยังมีไพ่ ผู้เล่นต้องจั่วหนึ่งใบและเก็บไพ่ใบนั้นไว้นอก rack ระหว่างเดา โดยต้องเดาอย่างน้อยหนึ่งครั้งก่อนจึงจะวางไพ่ได้ ไพ่ปกติเดาด้วยหน้าและสี ส่วน Joker เดาด้วย `JOKER` โดยไม่มีสี
  - เดาถูก → เปิดไพ่เป้าหมายถาวร จากนั้นเลือกเดาต่อหรือหยุดก็ได้ ถ้าหยุด ผู้เล่นเลือกวางไพ่ที่จั่วในตำแหน่งที่เซิร์ฟเวอร์อนุญาต ไพ่ใบนั้นยังคว่ำอยู่ แล้วจบเทิร์น
  - เดาผิด → เปิดไพ่ที่จั่ว ให้ผู้เล่นเลือกวางในตำแหน่งที่เซิร์ฟเวอร์อนุญาต แล้วจบเทิร์น ไพ่เป้าหมายที่เดาผิดยังคว่ำอยู่
  - เซิร์ฟเวอร์ต้องตรวจตำแหน่งวางทุกครั้ง ไพ่ปกติวางได้เฉพาะตำแหน่งที่ยังเรียงตามหน้าและแดงก่อนดำ ส่วน Joker วางได้ทุกตำแหน่ง ไพ่ที่หน้าและสีเท่ากันอาจมีตำแหน่งถูกต้องได้มากกว่าหนึ่งตำแหน่ง
- เมื่อกองจั่วหมด ผู้เล่นถูกบังคับให้เดาเพียงหนึ่งครั้ง เดาถูกจะเปิดเป้าหมายและจบเทิร์นโดยไม่เสียไพ่ เดาผิดจะเข้าสู่ขั้นเลือกไพ่คว่ำของตนเองหนึ่งใบเพื่อเปิด แล้วจบเทิร์น
- Host ตั้งเวลา action ได้เป็น ปิด, 30, 60, 90, 120, 180 หรือ 300 วินาที ทุก action ที่สำเร็จจะรีเซ็ต deadline เพื่อให้การตัดสินใจถัดไปได้เวลาเต็ม Client แสดงเวลานับถอยหลังและเปลี่ยนเป็นคำเตือนสีส้มใน 10 วินาทีสุดท้าย หากไม่ทำ action ก่อนหมดเวลา ผู้เล่นจะแพ้ทันที: เซิร์ฟเวอร์เปิดไพ่ทั้ง rack รักษาและเปิดไพ่ที่จั่วค้างไว้ เลื่อนไปยังผู้เล่นถัดไป และตรวจผู้ชนะใหม่ ช่วงวาง Joker เริ่มต้น เจ้าของทุกคนที่ยังวาง Joker ไม่ครบเมื่อหมดเวลาจะถูกกำจัด ตัวจับเวลาหยุดระหว่างรอ reconnect
- Joker คือไพ่ Cipher ที่วางตรงไหนก็ได้ การเดา Joker ถูกจะเปิดเฉพาะ Joker ใบที่เลือกและให้เดาต่อเหมือนไพ่ปกติ
- ผู้เล่นแพ้เมื่อไพ่ทุกใบใน rack ถูกเปิด ผู้เล่น active คนสุดท้ายเป็นผู้ชนะ

## 2. Lobby และการเชื่อมต่อกลับ

- ห้องกำหนดจำนวนผู้เล่นรวม `desiredPlayers` ตั้งแต่ 3–6 ผู้สร้างห้องเป็น host และถือว่าพร้อมโดยอัตโนมัติเพราะเป็นผู้กด Start เกมไม่เริ่มอัตโนมัติ ผู้เล่นจริงที่ไม่ใช่ host ทุกคนต้องกด ready ส่วน host เริ่มเกมคนเดียวหรือพร้อมผู้เล่นจริงคนอื่นได้ ที่นั่งว่างจนถึง `desiredPlayers` จะถูกเติมด้วยบอทที่ควบคุมโดยเซิร์ฟเวอร์ การแก้ settings จะล้างสถานะ ready ของผู้เล่นที่ไม่ใช่ host
- ถ้า host ออกก่อนเริ่มเกม ให้โอน host ตามลำดับการเข้าห้อง
- หน้า lobby ใช้ flow เดียวคือ Create Room แล้วเลือกจำนวนผู้เล่นรวม ห้องส่วนตัวแสดงรหัส 6 หลักและลิงก์ชวน ส่วน Public Quick Play ไม่ต้องใช้รหัส
- บอททำ action ผ่าน transition ฝั่งเซิร์ฟเวอร์เท่านั้น ทั้งเลือกไพ่เริ่ม วาง Joker จั่ว เดา วางไพ่ และรับโทษ โดยใช้ validation การเรียงไพ่ เทิร์น การเปิดไพ่ และแพ้ชนะชุดเดียวกับผู้เล่นจริง
- Host เลือกบอทระดับ Easy, Normal หรือ Hard ได้ โดย Easy เดาสุ่ม, Normal ไม่เดาคำตอบที่เคยเห็นว่าผิดซ้ำกับไพ่ใบเดิม และ Hard ใช้ไพ่ข้างเคียงที่เปิดแล้วกับกฎการเรียง rack เพื่อลดตัวเลือก บอททุกระดับห้ามอ่านค่าของไพ่คว่ำคู่แข่ง
- ห้องส่วนตัวมีรหัส 6 หลักและลิงก์ชวนที่คัดลอกได้ โดยยังคงมี Public Quick Play และ Create Room แบบปัจจุบัน
- เซิร์ฟเวอร์ส่ง activity log สาธารณะแบบจำกัดจำนวนและสถิติการเดาของแมตช์ เมื่อจบเกมจะแสดงหน้าผล และ Host ขอ Rematch เพื่อกลับสู่ lobby พร้อมสร้างแมตช์ใหม่ได้
- ผลของผู้เล่นที่ลงทะเบียนถูกส่งจาก realtime server ไป FastAPI โดยตรง เพื่อเก็บประวัติล่าสุด อัตราชนะ ความแม่นยำ และ streak ส่วน Guest ไม่ถูกบันทึก
- Client มี tutorial แบบแนะนำ, หน้ารอ reconnect พร้อมนับถอยหลัง, เสียง, Reduce motion, High contrast, สัญลักษณ์ช่วยแยกสี, ปรับขนาดไพ่, เปลี่ยนภาษา และเลือกธีม
- ล็อกห้องเมื่อ host เริ่ม phase เลือกไพ่คนเริ่มและไม่รับผู้เล่นกลางเกม
- เมื่อผู้เล่นหลุด ให้หยุด action ทั้งห้องและรอเชื่อมต่อกลับ 30 วินาที
- ถ้าไม่กลับภายในเวลา ให้เปิดไพ่ทั้งหมดของผู้เล่นคนนั้นและแพ้โดย forfeit ถ้ามีไพ่ที่จั่วแต่ยังไม่วาง ต้องนำไปวางและเปิดก่อนเพื่อไม่ให้ไพ่หายจาก authoritative state
- ถ้าเป็นเทิร์นของผู้เล่นที่ forfeit ให้เลื่อนไปผู้เล่น active คนถัดไป และตรวจเงื่อนไขผู้ชนะทุกครั้ง

## 3. แนวทางต่อยอด

รายการต่อไปนี้เลื่อนไปทำหลังระบบเล่นคนเดียว/3–6 คนปัจจุบันเสร็จและเสถียร และยังไม่ใช่กติกาที่นำมาใช้จริง:

- เพิ่มไพ่ Wild/Cipher ที่มีความสามารถพิเศษ
- เพิ่มโหมดทีม 2v2 พร้อม hint token ร่วมกัน
- เพิ่ม ranked/MMR matchmaking

## 4. Tech stack

### Phase 1 — Web multiplayer

- Frontend: React + TypeScript + Vite
- Realtime: Node.js + Colyseus แบบ authoritative server
- Backend สำหรับ auth/leaderboard: FastAPI
- ฐานข้อมูล: PostgreSQL และ Redis
- Deploy: Docker Compose บน Proxmox/Docker และ reverse proxy ภายนอก

### Phase 2 — Desktop

- ห่อ client ด้วย Electron หรือสร้างใหม่ด้วย Godot 4
- เพิ่ม Steamworks หลัง web core loop พร้อมใช้งานเท่านั้น
- Realtime backend และ protocol เดิมยังใช้ต่อได้

## 5. Roadmap โดยย่อ

1. ทดสอบ local ด้วย browser 3–6 session
2. เชื่อม Colyseus ผ่าน LAN และ internet
3. ทำ matchmaking, match history และ leaderboard
4. ขัดเกลา UI/UX, เสียง และ animation
5. ทำ desktop build และทดสอบ platform integration
